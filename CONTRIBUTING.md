# Contributing Guidelines

Thanks for contributing to Chat3D. This guide explains how to report issues and submit pull requests.

## Reporting bugs and feature requests

Use GitHub Issues and include:
- Repro steps
- Expected vs actual behavior
- Relevant environment details
- Logs/screenshots when useful

## Pull requests

Before opening a PR:
1. Rebase on the latest `dev` branch.
2. Keep changes focused; avoid unrelated formatting churn.
3. Run local tests for changed packages.
4. Use clear commit messages.

PR checklist:
1. Link the related issue (if any).
2. Explain behavior changes and risk.
3. Include test evidence.
4. Resolve CI failures before requesting review.

## Security reporting

Do not open public issues for security vulnerabilities.
Report vulnerabilities privately to project maintainers.
